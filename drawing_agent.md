# 그림 에이전트 (Drawing Agent) 가이드

Scan Voca 블로그 글에 들어가는 AI 일러스트의 규칙을 정리한 운영자 문서입니다.

## 동작 방식

1. 관리자 페이지(`/admin/blog`)에서 글 초안을 편집한 뒤 **[이미지 계획 세우기]**를 누르면,
   AI가 본문을 읽고 "어느 위치에 어떤 그림"을 넣을지 계획을 제안합니다.
2. 이미지 개수는 고정이 아니라 **문맥에 맞게 0~5개**가 제안됩니다. 대표 이미지(hero)는 최대 1개이며
   블로그 목록의 썸네일로도 쓰입니다.
3. 계획은 항목별로 **포함/제외, 장면 설명 수정, 직접 추가**가 가능합니다. 승인한 항목만 생성됩니다.
4. 생성된 이미지는 미리보기에서 확인하고, 마음에 안 들면 항목별 [다시 생성]. 
5. [본문에 반영] 후 최종 미리보기까지 확인해야만 게재됩니다. **검토 없이 게재되는 것은 없습니다.**

## 스타일 규칙 (모든 그림에 자동 적용)

실제 규칙의 단일 소스는 `server/app/services/image_style.py`의 `IMAGE_STYLE_GUIDE` 상수(영문)입니다.
아래는 그 한국어 요약입니다:

- 밝고 친근한 **플랫 벡터 일러스트** (부드러운 곡선, 가벼운 명암)
- 포인트 컬러: **indigo(#4F46E5) · violet(#7C3AED)** + 파스텔 보조색 (서비스 UI와 통일)
- 배경: 깨끗한 흰색/아주 밝은 톤, 복잡한 배경 금지
- 비율: 16:9 권장 (블로그 헤더/본문 그림에 적합)
- 인물: 한국 학생·직장인을 자연스럽고 따뜻하게
- **이미지 안에 글자·숫자·문자 절대 금지** — AI가 글자를 그리면 대부분 깨져 보입니다.
  책·화면·간판은 빈 페이지나 추상 도형으로 대체
- 워터마크·로고·브랜드명 금지

> ⚠️ 모델이 "글자 금지" 규칙을 가끔 어깁니다(책 표지에 외계문자 등).
> 미리보기에서 깨진 글자가 보이면 [다시 생성]을 누르세요.

## 스타일을 바꾸고 싶을 때

1. `server/app/services/image_style.py`의 `IMAGE_STYLE_GUIDE` 문구를 수정
2. 백엔드 재배포 (`cd server && .\deploy-final.ps1`)
3. 이 문서의 한국어 요약도 함께 갱신

## 기술 정보

- 이미지 생성 모델: `gemini-2.5-flash-image` (Google AI API, 기존 GEMINI_API_KEY 사용)
- 비용: 이미지 1장당 소액 과금 (글당 2~3장 기준 무시할 수준)
- 저장 위치: `web/public/blog-images/{글slug}/{번호}.png` — 글과 함께 git 저장소에 커밋됨
- 수요 급증 시 생성이 일시 실패(503)할 수 있음 → 잠시 후 [다시 생성]으로 해결
