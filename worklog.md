# scan voca 프로젝트 작업 로그 사용 가이드 📝

이 문서는 작업 로그 관리 방법만을 안내합니다. 실제 작업 기록은 `WORKLOG(1).md`, `WORKLOG(2).md`, `WORKLOG(3).md` ... 파일에 누적합니다.

## 작성 원칙
- 날짜순(오래된 것 → 최신)으로 기록합니다.
- 하루 단위 섹션에 "핵심 성과/문제/원인/해결/결과" 형식으로 간결히 적습니다.
- 코드/명령은 최소화하고, 중요한 파일 경로만 명시합니다.
- **커밋 기반 작업 기록**: 커밋 후 `git log` 및 `git diff`로 실제 변경사항을 파악하여 작성합니다.

## 파일 분할 규칙
- 한 파일 길이가 약 600줄에 도달하면 새 파일을 만듭니다.
- 파일명은 `WORKLOG(n).md`이며 숫자는 1부터 증가합니다.
- 가장 오래된 기간은 `WORKLOG(1).md`, 이후 순차적으로 `WORKLOG(2).md`, `WORKLOG(3).md` ... 에 기록합니다.

## 운영 방법(순서)
1. 최신 파일을 엽니다. 없으면 `WORKLOG(1).md`부터 시작합니다.
2. 오늘 날짜 섹션을 추가하고 작업 내용을 기록합니다.
3. 파일이 600줄을 넘기면 다음을 수행합니다.
   - 새 파일 `WORKLOG(k+1).md`를 생성(헤더: 기간 범위만 기입 가능)
   - 기존 파일 하단에 다음 안내를 업데이트합니다.
     - “이전 파일” 링크: 직전 파일명
     - “다음 파일” 링크: 새 파일명
4. 이 가이드(`WORKLOG.md`)는 수정하지 않습니다. 사용법만 유지합니다.

## 링크 규칙
- 각 `WORKLOG(n).md` 하단에 다음을 유지합니다.
  - 이전 파일: `WORKLOG(n-1).md` (n>1인 경우)
  - 다음 파일: `WORKLOG(n+1).md` (존재할 때만, 없으면 “600줄 도달 시 생성” 문구)

## 예시
- `WORKLOG(1).md`: 2024-06-13 ~ 2025-08-04
- `WORKLOG(2).md`: 2025-08-05 ~ 2025-08-13
- `WORKLOG(3).md`: 600줄 도달 시 생성
