## 변형 단어 기본형 변환 문제
- **문제**: words, expressions 등 변형된 단어가 기본형(word, expression)으로 변환되지 않음
- **시도한 방법들**:
  1. GPT 프롬프트에 기본형 변환 지시 추가 → GPT는 기본형 반환
  2. GPT 응답 후 로컬 DB 재확인 로직 추가 → 캐시 때문에 GPT가 호출되지 않음
  3. 클라이언트 측 기본형 추론 함수 추가 → 복잡도만 증가, 작동 안 함
  4. 캐시 검증 로직 추가 (변형 그대로 캐시되면 삭제) → 정상 캐시까지 삭제됨
- **결과**: 오히려 상황 악화 (active, dessert 등 정상 단어의 캐시가 삭제되어 레벨4로 재생성)
- **현재 상태**: 복잡한 로직 전부 롤백, 원래대로 복구
- **근본 원인**:
  - 이미 캐시에 잘못 저장된 데이터 (words → words 그대로)
  - 캐시가 있으면 GPT 호출 안 되어 기본형 변환 로직이 작동하지 않음
- **다음 담당자를 위한 조언**:
  1. 캐시 초기화부터 시작 (AsyncStorage 완전 삭제)
  2. GPT 프롬프트의 기본형 변환 지시는 유지 (이미 추가됨)
  3. 클라이언트 측 복잡한 추론 로직은 추가하지 말 것
  4. 단순하게: GPT가 반환한 기본형 → 로컬 DB 재확인 → DB 레벨 사용 (이 로직은 이미 구현됨)

## DB 확인 결과 (참고)
- active: DB에 없음 → 레벨4 맞음
- dessert: DB에 없음 → 레벨4 맞음
- word: DB에 있음 (Lv.1)
- expression: DB에 없음
