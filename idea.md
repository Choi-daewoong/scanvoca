# 게시판(커뮤니티) 기능 브레인스토밍

> 작성일: 2026-06-14
> 상태: 아이디어 단계 (구현 전)

---

## 1. 배경 / 목표

- 공지사항을 게시할 공간이 필요함 (업데이트 안내, 점검 안내 등)
- 학생들끼리 단어장을 공유하면:
  - 스캔량 감소 (서버 비용/AI 호출 절감과도 직결)
  - 사용자 입장에서도 "스캔 → 검토 → 저장"보다 "코드 입력 → 즉시 단어장 확보"가 훨씬 빠름
  - 같은 교과서/문제집을 쓰는 학생들이 모이는 자연스러운 커뮤니티 효과 → 재방문율 상승

기존에 이미 **단어장 공유 코드(8자리, 영숫자, 혼동 문자 제외)** 기능이 구현되어 있음
(`server/app/services/wordbook_service.py` `SHARE_CODE_ALPHABET`/`SHARE_CODE_LENGTH = 8`).
→ 게시판은 이 기존 기능을 "발견 가능하게" 만드는 레이어로 보면 됨. 코드 자체는 새로 만들 필요 없음.

## 2. 게시판 탭 구조 (제안)

네비게이션에 새 탭 추가. 현재 `BottomNav`는 홈/스캔/단어장/설정 4개 탭 → 5개가 되면 모바일에서 다소 빡빡함.

**옵션 A**: 5번째 탭으로 "게시판" 추가 (아이콘만 작게 조정)
**옵션 B**: "설정" 탭을 "더보기"로 바꾸고 그 안에 설정/게시판을 묶기
**옵션 C**: 상단에 별도 탭 바 추가 (하단 네비는 핵심 4개만 유지)

→ 개인적으로는 **옵션 A**가 가장 직관적이라고 생각함. 공유 게시판이 핵심 기능이 될 수 있어서 한 탭 깊이 더 들어가면 사용률이 떨어질 가능성이 큼.

### 게시판 하위 탭 (가로 스크롤 or 세그먼트 컨트롤)
1. **공지사항** — 운영진(관리자)만 작성, 사용자는 읽기만
2. **단어장 공유** — 메인 기능 (아래 3번에서 상세)
3. **Q&A** — 사용자 질문 + 답변 (운영진 또는 다른 사용자)
4. **FAQ** — 자주 묻는 질문 (정적 또는 관리자 큐레이션)

## 3. 단어장 공유 게시판 (핵심 기능)

### 3.1 기본 플로우
- 사용자가 단어장 상세에서 "게시판에 공유하기" 버튼 클릭
- 게시글 작성: 제목, 설명, 카테고리/태그, (자동으로 share_code 첨부)
- 다른 사용자는 게시글을 보고 **"내 단어장에 추가" 버튼 한 번**으로 바로 가져오기
  → 기존 "공유 코드 입력" 플로우보다 마찰이 훨씬 줄어듦

### 3.2 메타데이터 / 분류 (검색·필터 핵심)
학생들이 찾는 기준은 결국 "어떤 교재의 몇 단원인가"이므로:
- **과목/교재 구분**: 교과서(출판사+학년), 모의고사/수능, 토익/토플, 자체 제작
- **단원/범위**: 자유 텍스트 (예: "능률 영어1 3단원", "고2 3월 모의고사")
- **단어 수** (자동 계산, 노출)
- **태그**: 자유 태그 여러 개 (예: #고1, #천재교육, #모의고사)

→ Wordbook 또는 별도 `SharedWordbookPost` 테이블에 이 메타데이터 저장.
검색은 태그/제목 기반 단순 LIKE 검색으로 MVP는 충분.

### 3.3 신뢰도 / 발견성 보조 기능
- **가져오기 수(=복제 횟수)** 카운트 → 인기순 정렬
- **좋아요/스크랩**
- **댓글**: "오타 있어요", "이 단원 단어 맞아요" 같은 품질 피드백 → 게시자가 원본 수정 시 활용
- **미리보기**: 게시글에서 단어 목록 일부(예: 5~10개)를 펼쳐볼 수 있게 — 가져오기 전에 퀄리티 확인
- **신고 기능**: 부적절/저품질/도배 게시글 신고 → 운영진 처리

### 3.4 "구독" / 업데이트 동기화 (확장 아이디어)
- 원본 단어장이 업데이트(단어 추가/수정)되면, 가져간 사용자에게 "업데이트 있음" 알림
- 사용자가 선택적으로 "최신화" 가능 (단, 본인이 수정한 뜻은 덮어쓰지 않도록 주의)
- 이 부분은 복잡도가 높아서 **MVP 이후 단계**로 분류 추천

### 3.5 어뷰징 방지
- 동일 단어장 share_code로 게시글 중복 생성 방지 (또는 허용하되 원본 링크만 다르게)
- 게시글 작성/가져오기에 대한 rate limit (이미 OCR 등에 RateLimiter 있음 → 동일 패턴 재사용 가능)
- 신고 누적 시 자동 숨김 임계치

## 4. Q&A / FAQ

### Q&A
- 단어 의미, 학습법, 앱 사용법 등 자유 질문
- 답변은 다른 사용자 또는 운영진이 작성
- "채택" 기능(질문자가 가장 도움된 답변 선택)은 초기엔 생략 가능 — 추천수(좋아요)로 충분

### FAQ
- 처음에는 정적 페이지(하드코딩 또는 마크다운)로 시작 — DB/CMS 불필요
- 비밀번호 재설정, 공유 코드 사용법, OCR 인식 안 될 때 등 운영 중 반복되는 문의를 수집해서 채워나가는 방식
- 추후 검색 많이 되는 키워드는 Q&A → FAQ로 승격

### 공지사항
- 관리자 권한(`User.role` 또는 `is_admin` 플래그) 필요 — 현재 User 모델에 없으면 추가 필요
- 새 공지 등록 시 "안 읽음" 배지를 홈 또는 게시판 탭에 표시 (간단한 `last_read_at` 비교로 구현 가능)

## 5. 기술적 고려사항 (개략)

### 새 모델 (가안)
- `posts` (id, board_type[notice/qna/share], author_id, title, content, wordbook_id(nullable), tags, view_count, like_count, import_count, created_at, ...)
- `comments` (id, post_id, author_id, content, created_at)
- `post_likes` / `post_reports` (id, post_id, user_id)
- `users.role` (admin/user) — 공지사항 작성 권한 체크용

### API
- `GET /api/v1/board/posts?board_type=share&tag=...&sort=popular`
- `POST /api/v1/board/posts` (단어장 공유 시 wordbook_id 첨부)
- `POST /api/v1/board/posts/{id}/import` → 내부적으로 기존 share_code 가져오기 로직 재사용
- `POST /api/v1/board/posts/{id}/comments`, `/like`, `/report`

### 기존 자산 재사용
- 단어장 가져오기 자체 로직은 `get_by_share_code` + 기존 "단어장 복제" 로직을 그대로 호출하면 됨
- 신규 게시판은 사실상 "share_code에 대한 메타데이터 + 발견(검색/정렬) 레이어"

## 6. MVP 우선순위 제안

1. **공지사항** (가장 단순, 관리자 1명만 작성 — DB 모델 1개 + 목록/상세 페이지)
2. **단어장 공유 게시판** (핵심 가치, 기존 share_code 활용해 마찰 최소화)
   - MVP 범위: 게시글 작성/목록/검색(태그)/가져오기 버튼만
   - 좋아요/댓글/구독은 2단계
3. **FAQ** (정적 페이지로 빠르게 추가 가능)
4. **Q&A** (커뮤니티 활성화는 시간이 걸리므로 우선순위 가장 낮음 — 초기엔 운영진이 직접 응대하는 게 더 효율적일 수도)

## 7. 추가로 떠오르는 아이디어

- **학교/학년 프로필**: 회원가입 시 학년/사용 교과서 선택 → 공유 게시판에서 "내 학년/교과서" 단어장 우선 노출
- **단어장 공유 시 포인트 보상**: 로드맵에 있는 게이미피케이션(스트릭/별/랭킹)과 연계 — 많이 공유되고 가져가질수록 포인트 적립
- **주간 인기 단어장**: 홈 화면에 "이번 주 인기 공유 단어장" 위젯 노출 → 게시판 유입 경로 확보
- **출처 표기**: 가져온 단어장에 "OO님이 공유한 단어장에서 추가됨" 같은 출처 메타 유지 (저작권/품질 추적용)
- **시험 기간 캘린더 연동**: "중간고사 D-7, 이 교과서 단어장 모음" 같은 시즌성 노출 — 약간 과한 기능이라 후순위
